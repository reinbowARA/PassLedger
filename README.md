# PassLedger

PassLedger — это безопасный менеджер паролей с графическим интерфейсом, разработанный на Go с использованием библиотеки Fyne. Приложение позволяет хранить и управлять паролями в зашифрованной базе данных, защищенной мастер-паролем.

## Функции

- **Защищенное хранение**: Все пароли шифруются с использованием мастер-пароля и хранятся в локальной SQLite-базе данных.
- **Управление записями**: Добавление, редактирование и удаление учетных записей (название, логин, пароль, URL, заметки).
- **Организация по группам**: Группировка записей для удобного управления.
- **Поиск**: Быстрый поиск по записям.
- **Копирование паролей**: Встроенная функция копирования паролей в буфер обмена.
- **Кроссплатформенность**: Работает на Windows, Linux и macOS.

## Требования

- Go 1.24.5 или выше
- Зависимости автоматически устанавливаются через `go mod`

## Установка

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/reinbowARA/PassLedger.git
   cd PassLedger
   ```

2. Скачайте зависимости:
   ```bash
   go mod download
   ```

## Сборка

Используйте Makefile для сборки:

- **Стандартная сборка** (для текущей платформы):
  ```bash
  make build
  ```
  Исполняемый файл будет создан в `build/passledger`.

- **Сборка для Windows**:
  ```bash
  make build-win
  ```
  Исполняемый файл: `build/passledger.exe`.

- **Сборка для Linux**:
  ```bash
  make build-linux
  ```
  Исполняемый файл: `build/passledger`.

- **Сборка для macOS**:
  ```bash
  make build-mac
  ```
  Исполняемый файл: `build/passledger`.

## Запуск

Запустите приложение командой:
```bash
make run
```
или выполните собранный бинарный файл.

При первом запуске приложение предложит создать мастер-пароль. В дальнейшем используйте его для доступа к базе данных.

## Использование

1. **Вход**: Введите мастер-пароль для доступа к данным.
2. **Добавление записи**: Нажмите кнопку "Добавить" и заполните поля (название, логин, пароль, URL, заметки).
3. **Управление группами**: Создавайте и редактируйте группы для организации записей.
4. **Поиск**: Используйте поле поиска для фильтрации записей.
5. **Просмотр и копирование**: Выберите запись, чтобы просмотреть детали и скопировать пароль.

## Архитектура

Проект организован по модулям:

- `app/`: Графический интерфейс (окна, формы).
- `crypto/`: Функции шифрования и хэширования.
- `db/`: Взаимодействие с базой данных SQLite.
- `models/`: Структуры данных.

## Безопасность

Для шифрования используются криптографические функции Go (golang.org/x/crypto) и gost (github.com/pedroalbanese/gogost).

- Мастер-пароль используется для генерации ключа шифрования через PBKDF2.
- Данные шифруются с помощью российских гостов такие как:
  - Симметричный алгоритм блочного шифрования ГОСТ 34.12-2018 (Кузнечик),
  - Хеш-функцией ГОСТ 34.11-2012 (Стрибог).
- База данных хранится локально, доступ которого возможен только через мастер-пароль.
